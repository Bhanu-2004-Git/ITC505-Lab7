Addendum – Process & Design Explanation

This Mad Lib webpage was developed using basic HTML, CSS, and JavaScript. The goal was to generate a simple, interactive story without using any server-side code such as ExpressJS POST routes. Instead, all form data is processed directly in the browser using JavaScript.

When the user fills in the five text fields—name, place, animal, food, and hobby—and clicks the “Create Story” button, an event listener intercepts the form submission, prevents the default page reload, and dynamically builds a short story using the values entered. The result is then displayed on the same page inside a styled result box.

The page design emphasizes clarity and readability through centered layout, soft borders, and responsive spacing. A footer script automatically shows the last modified date of the file, satisfying the lab’s submission requirements. This approach demonstrates how a small amount of JavaScript can provide interactive behavior even without the need for a backend server or database connection.
